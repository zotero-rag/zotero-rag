## Guidelines

### Code and Pseudo-code Guidelines

Keep code to a minimum. Only use pseudo-code to describe novel algorithms, or link to external prototypes that prove the implementability of the design.

### Verbiage

Prefer a relatively formal tone. An informal is fine, but keep it consistent. In-jokes or other humorous comments are allowed as long as they do not detract from the main content.
